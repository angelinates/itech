Индивидуальное домашнее задание

Индивидуальное домашнее задание (ИДЗ) является одним из видов работ в рамках курса Internet-технологии (1-й семестр).
Минимальная оценка (10 баллов) будет автоматом выставлена при отработке всех лабораторных работ, но без выполнения самого ИДЗ. Этот минимум может быть поднят путём собственно реализации задания (т.е. выполнения ИДЗ) и получения вместо минимальных 10 баллов чего-то более высокого.
Оценивается выполненное ИДЗ от 10 до 18 баллов (если не указано иного ограничения максимальной оценки) на основании его сдачи (отдельных понятий "отработки" и "защиты" в контексте ИДЗ нет). Сдача ИДЗ заключается в демонстрации выполненного задания, его исходного кода и ответах на вопросы по ходу проверки.
Выбор варианта задания на своё усмотрение из предложенных. В отличие от допзаданий, согласовывать этот выбор не надо. Выбрали, уточнили, если что-то непонятно, сделали, показали (на лабораторной или консультации).
Крайний срок сдачи ИДЗ - день выставления зачета.
При выполнении заданий помните, что гуглом преподаватель тоже пользоваться умеет (да и работы других студентов неплохо помню, благо, ИДЗ сдают достаточно редко, из-за наличия автобаллов), и недобросовестное повторное использование познакомит с новыми интересными вопросами.

Доступны следующие темы индивидуальных заданий, на ваш выбор:

JS, Букмарклет, Хабр.
Букмарклет для обработки страниц статей с ресурса habr.com. Удаление боковой, верхней и нижней панелей, кнопки "Наверх" (возникающей при прокрутке), скриптов гугл-аналитики и т.д. - т.е. тех элементов страницы, которые не нужны для отображения собственно ее контента. Обратите внимание на различия в структуре страниц статей обычных пользователей и корпоративных блогов.
Максимальная оценка: 14 баллов.

JS, Букмарклет, SO.
Букмарклет для обработки страниц с вопросами/ответами с ресурса stackoverflow.com, пример типовой обрабатываемой страницы. Задача букмарклета - удаление ненужных пользователю элементов страницы (футер, левый сайдбар, лишние скрипты и т.п.) и выделение (цветом, фоном или т.п.) принятого ответа, если он есть.
Максимальная оценка: 14 баллов.

JS, Онлайн-таймер.
Приложение на JavaScript для оповещения пользователя с заданной периодичностью. Отображение оставшегося до оповещения времени на странице и в заголовке окна. Возможность указания периода работы, запуска, приостановки, отмены таймера и настройки действия после завершения отсчета (прекращение работы или повторный запуск таймера). Оповещение через Notification API. Возможность управления таймером не только через элементы интерфейса на странице, но и через параметры GET-запроса (т.е. параметры работы таймера - периодичность и другие - могут быть указаны в адресной строке, что открывает интересные сценарии использования).
Максимальная оценка: 16 баллов.

JS, Игра, Крестики-нолики.
Два режима игры: многопользовательский и однопользовательский. В многопользовательском игра ведётся двумя игроками в одном приложении, поочерёдно. В однопользовательском потребуется реализация логики несложного "бота", отвечающего на ходы игрока. Предусмотреть возможности: сброс игрового поля (начало игры), отображение очередности текущего хода, ведение счетчика игр и их результатов.
Максимальная оценка: 15 баллов.

JS, Игра, 15.
Описание игры доступно по ссылке. Предусмотреть возможность перемешивания костяшек в начале игры.
Максимальная оценка: 17 баллов.

JS, Игра, Змейка.
Классическая игра на JavaScript, с использованием canvas.
Максимальная оценка: 18 баллов.

JS, Игра, Гонки.
Приложение с использованием canvas. Двумерная "бесконечная дорога", по которой перемещается автомобиль игрока и автомобили NPC. Управление перемещением автомобиля игрока влево-вправо (для избегания столкновения с автомобилями NPC, случайно появляющимися по дороге) клавишами клавиатуры. При столкновении с NPC игра завершается. Ведение счета очков, в качестве которых логично использовать пройденный игроком "километраж" до завершения игры.
Максимальная оценка: 18 баллов.

JS, Игра, Breakout.
Игра на JavaScript, с использованием canvas. Описание игры доступно по ссылке.
Максимальная оценка: 18 баллов.

JS, Игра, Жизнь.
Иллюстрация работы клеточного автомата - развитие клеток на поле, причем ход развития задан параметрами, установленными пользователем еще до начала игры. Классика жанра Zero Player Game (с приличной наукоёмкостью, впрочем, это уже за рамками нашего курса). Описание игры доступно по ссылке.
Максимальная оценка: 18 баллов.

Статический веб-ресурс (HTML/CSS/JS), на произвольную тематику (описание игры или фильма, фан-сайт музыкальной группы или футбольной команды, портфолио своей команды разработчиков, меню пиццерии, описание города, программного обеспечения, учебного пособия, проблем животноводства и т.д.), созданный с использованием инструмента gulp (с плагинами для CSS-препроцессоров и минификации). Наличие краткого ТЗ к проекту (включающего описание задач ресурса, указание целевой аудитории, существующие аналоги и их плюсы/минусы, предполагаемый функционал и используемые средства разработки) обязательно. Формат ТЗ произволен, для него логично использовать файл README.md репозитория с проектом. При проверке, в том числе, будет учитываться соответствие контента заявленной в ТЗ тематике и адекватность используемых технологий решаемым задачам.
Максимальная оценка: 18 баллов.

PHP, CLI утилита.
Задача на знакомство с использованием PHP-скриптов в консоли (PHP CLI). Утилита для распределения файлов фотографий по папкам, в соответствии с данными из EXIF о дате снимка.
Максимальная оценка: 15 баллов.

PHP, Библиотека.
Возможность добавления текстовых (или другого формата) файлов на ресурс, отображение списка уже имеющихся ранее загруженных файлов и их открытие.
Максимальная оценка: 16 баллов.

PHP, Блог.
Пользователь регистрируется (учетные данные хранятся в файле на сервере - полноценные БД пока использовать нельзя), после чего может добавлять записи, сохраняемые на сервере в текстовом файле, ассоциированном именно с этим пользователем. При входе пользователя отображаются прошлые сделанные им записи.
Максимальная оценка: 18 баллов.

PHP, Калькулятор расходов.
Возможность регистрации и авторизации пользователя, после авторизации пользователю отображается список его расходов (ранее внесённых им) и возможно добавление пользователем информации о своих новых расходах (название позиции и стоимость). Для хранения учётных данных пользователя и информации о его расходах использовать файлы на сервере (полноценные БД пока использовать нельзя), логично использовать отдельный файл для хранения данных о расходах каждого пользователя.
Максимальная оценка: 18 баллов.
