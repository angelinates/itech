Лабораторная работа №3

Язык создания сценариев JavaScript
3.1 Цель работы
  • Познакомиться с языком программирования JavaScript для создания интерактивных HTML-документов.
  • Рассмотреть применение библиотеки jQuery и средств Vue.js.
3.2 Описание лабораторной установки
    Для выполнения лабораторной работы рекомендуется использовать следующее свободное программное обеспечение:
  • редактор кода MS Visual Studio Code или любой другой, знакомый пользователю и имеющий подсветку синтаксиса (Notepad++, Sublime Text, Atom, Brackets и т.п.);
  • браузер Mozilla Firefox или Google Chrome;
  • СКВ Git (для работы с сервисом GitHub потребуется доступ к сети Интернет).
3.3 Порядок выполнения работы
  1. Создать HTML-документ с произвольным содержимым (если по заданию не предполагается написание букмарклета для заданного веб-ресурса).
  2. Добавить скрипт (или подготовить букмарклет) согласно варианту задания. Возможно применение библиотеки jQuery или средств Vue.js.
  3. Быть готовым к вопросам по коду.
  4. Проверить работу в разных браузерах.
  5. Внести в проект указанные преподавателем (при попытке получения отработки) изменения.
Задания для лабораторной являются индивидуальными, но при работе в составе бригады, возможно совместное выполнение в бригаде вариантов всех ее участников.
Номер варианта студента указан в электронном журнале. Выполнение чужого варианта является основанием для отклонения работы.
Отработка выставляется после совместной со студентом проверки работы проекта, выполненного в соответствии со всеми заданиями этого раздела.
3.4 Варианты заданий

  1. В папке images ресурса находятся изображения 1.jpg, 2.jpg, 3.jpg, 4.jpg, 5.jpg, 6.jpg, 7.jpg, 8.jpg, 9.jpg (это для примера - просто должно быть некоторое множество изображений). На странице выводится первое изображение из этой папки и присутствует выпадающий список, содержащий соответствующие рисункам пункты, позволяющий отобразить рисунок согласно выбору пользователя. Предоставить пользователю возможность изменять параметры изображения (ширину и высоту изображения, толщину рамки, альтернативный текст).

  2. Имеется страница с данными о температуре. Написать скрипт (можно в виде букмарклета или в виде скрипта в составе предложенной страницы), выводящий информацию о самом жарком дне (его дату и соответствующую температуру).

  3. На странице представлена галерея рисунков (несколько изображений, размещённых вертикально) и кнопки "выше" и "ниже" для изменения их порядка (при нажатии на эти кнопки смещается выбранное изображение). Способ выбора изображения - на своё усмотрение. Предоставить пользователю возможность изменять параметры изображения (ширину и высоту изображения, толщину рамки, альтернативный текст).

  4. Реализовать смену изображений на странице через заданный пользователем (с помощью модального окна при загрузке страницы) интервал. Оформление страницы (например, фон) должно меняться в зависимости от текущего времени суток.
    Написать скрипт для создания таблицы 10 на 10, заполненной сгенерированными случайными целыми числами в диапазоне, указанном пользователем в полях для ввода числовых значений. Оформление таблицы (фон ячеек) по подобию шахматной доски.

  5. Написать скрпит, случайным образом генерирующий данные и выводящий их пользователю в виде таблицы, в которой постепенно добавляются новые строки. Должно отличаться оформление четных и нечетных строк данных, а также первой строки таблицы (заголовочной). Перед началом вывода пользователь может указать интервал, с которым появляются новые строки с данными. Предоставить возможность останавливать генерацию данных.

  6. Приложение, предполагающее указание данных студентов (оценки по предметам) и их анализ (определение числа студентов в каждой из категорий: "отличник", "хорошист" и "неуспевающий"). В форме (анкете) указывается фамилия и пять оценок за дисциплины (от 2 до 5). "Отличник" ‐ если все оценки "5", "хорошист" ‐ оценки "5" и "4", "неуспевающий" ‐ хотя бы одна оценка "2". Необходимо последовательно указать данные для произвольного числа студентов (последовательное заполнение формы данными для разных студентов ‐ внесли для одного, учли, внесли для следующего, учли, и т.д.), после чего вывести сводку о количестве студентов в каждой из категорий и перечислить фамилии неуспевающих. Для упрощения отладки рекомендуется (но для отработки не обязывает!) добавить возможность не заполнять оценки вручную, а использовать генератор случайных чисел.

  7. На странице находится набор полей для ввода данных о пяти сотрудниках ‐ фамилии и дата приема на работу. Рассчитать и вывести пользователю средний стаж работы на текущее время, в виде целого числа лет.

  8. Калькулятор, представляющий из себя набор кнопок для ввода цифр и указания операций (сложение, вычитание, умножение, деление), поле для вычисляемого выражения (формируемого путём нажатия на соответствующие кнопки на странице), поле для вывода результата.

3.5 Содержание отчета
Отчет по лабораторной работе должен содержать: цель работы, вариант задания, исходный код веб-страницы, стилевых файлов и файлов скриптов, изображение полученного документа, выводы по работе. Наличие отчета не является необходимым.
