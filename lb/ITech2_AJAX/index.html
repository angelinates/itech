1. Мета роботи

Вивчити програмні механізми технології AJAX для динамічного надсилання або завантаження даних із сервера.

2. Методичні вказівки

Під час підготовки до виконання лабораторної роботи слід ознайомитися з особливостями синхронного та асинхронного методів передачі даних, вивчити формати обміну даними JSON та XML, а також повторити методи та властивості класу XMLHttpRequest, а також можливості Fetch API.

3.Опис лабораторної установки

Для виконання лабораторної роботи рекомендується використовувати:

  • текстовий редактор, що має підсвічування синтаксису: Notepad++ або Visual Studio Code;
  • браузер: Mozilla Firefox або Google Chrome;
  • складання веб-сервера: XAMPP або OpenServer.

Засоби, що рекомендуються, є вільним програмним забезпеченням.

Для виконання лабораторної роботи не потрібний доступ до мережі Інтернет.

4. Порядок виконання роботи
  1. Використовується вихідний код із лабораторної роботи, присвяченої роботі з даними за допомогою інтерфейсу PDO або MySQLi. Перевірити його роботу.
  2. Змінити код для обробки даних результатів запитів (для параметризованих запитів, що реалізуються в минулій лабораторній за варіантами) таким чином, щоб виведення результату користувачеві не призводило до перезавантаження сторінки - тобто, використовувати технологію Ajax. Використовувати такі формати відповіді від сервера:
    • у форматі простого тексту (безпосередньо отримувати згенерований фрагмент HTML-коду та виводити його в заданому елементі стартової сторінки);
    • у форматі XML (результат запиту на вибірку міститься у генерованому сервером XML-документі, який клієнт зчитує через властивість responseXml і формує висновок користувачеві);
    • в формате JSON (Результат запиту на вибірку поміщається в масив, який потім перетворюється на JSON-рядок за допомогою методу json_encode і відправляється клієнту, який отримує дані вибірки з рядка за допомогою методу JSON.parse і формує виведення користувачеві).
    Зверніть увагу, необхідно показати застосування всіх трьох форматів відповіді від сервера – по одному формату на кожен із трьох запитів на вибірку.
    Можливе використання замість XMLHttpRequest методів Fetch API для запиту, що є прикладом роботи з форматом JSON.
5. Зміст звіту
  • титульний лист із зазначенням теми лабораторної, прізвища, ініціалів та групи студентів, що виконали (до 3-х осіб)
  • мета роботи
  • варіант завдання
  • вихідні коди(HTML, JavaScript і РНР)
  • результати виконання асинхронних запитів у вигляді знімків екрана браузера
  • висновки щодо роботи

Допускається здавання звіту в електронному вигляді. Для захисту лабораторної наявність роздрукованого звіту не є необхідною. При викладанні вихідного коду на GitHub пам'ятайте про дамп БД, для тих варіантів використовуваної л/р PDO, в яких не пропонувалося вже готової БД для роботи.
