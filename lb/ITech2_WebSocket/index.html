1. Мета роботи

Дослідження програмних засобів для обміну повідомленнями між браузером та веб-сервером у режимі реального часу.

2. Методичні вказівки

Під час підготовки до виконання лабораторної роботи слід ознайомитися з механізмом обміну повідомленнями за протоколом WebSocket, а також подіями та методами бібліотеки Socket.IO для колективного обміну повідомленнями.

Протокол HTTP обмежений моделлю запит-відповідь: клієнт посилає HTTP-запит і чекає на нього HTTP-відповідь від сервера. Тобто. сервер не може повідомити щось клієнту (наприклад, передати свіжі повідомлення на форумі) доти, доки клієнт «у нього не спитає». WebSocket – протокол повнодуплексного зв'язку поверх TCP-з'єднання, призначений для обміну повідомленнями між браузером та веб-сервером у режимі реального часу. Завдяки можливості надсилання сервером даних клієнту без попереднього запиту у звичній схемі запит-відповідь між браузером і сервером може відбуватися рівноправний двосторонній (двонаправлений) обмін повідомленнями, поки з'єднання відкрито. Протокол WebSocket має значно менші накладні витрати за рахунок постійно відкритого каналу в порівнянні з HTTP-зверненнями, що повторюються, до сервера для відстеження змін. Організація такої взаємодії між браузером та веб-сервером дає можливість створювати інтернет-додатки з інтенсивним двостороннім обміном даними, вимогливими до швидкості обміну, наприклад, гри.

Перш ніж почати використовувати протокол WebSocket, потрібно створити веб-сервер, який підтримує цей протокол, наприклад, на основі серверного JavaScript-фреймворку Node.js. Для спрощення роботи може бути використана бібліотека Socket.IO, яка приховує від розробника низькорівневу роботу з передачею повідомлень.

Нижче наведено приклад створення програми чату. Передбачається, що платформа Node.js вже встановлена. У папці з вихідним кодом рекомендується створити файл package.json, що містить інформацію про проект, а головне, список залежностей - пакетів, необхідних для роботи програми, які можна завантажити за допомогою пакета npm. менеджер. Вміст файлу package.json:

{
  "name": "socket-chat-example",
  "version": "0.0.1",
  "description": "my first socket.io app",
  "dependencies": {
    "express": "^4.15.2",
    "socket.io": "^1.7.4"
  },
  "scripts": {
    "start": "node index.js"
  }
}

Для встановлення пакетів, визначених у package.json, використовується консольна команда span class = 'code_snippet' npm install /span , виконана в каталозі проекту. Код серверної частини слід розмістити у файлі index.js:

var app = require('express')();
var http = require('http').Server(app);
var io = require('socket.io')(http);

app.get('/', function(req, res) {
  res.sendFile(__dirname + '/index.html');
});

io.on('connection', function(socket){
  socket.on('send message', function(msg) {
	io.emit('receive message', msg);
  });
});

http.listen(3000, function() {
  console.log('listening on *:3000');
});

Щоб запустити сКод клиента размещен в файле index.html:ервер, використовуйте команду span class = 'code_snippet' node index.js /span , або span class = 'code_snippet' npm start /span . Сервер буде запущено за адресою http://localhost:3000. Під час доступу до нього буде виконано маршрутизацію, але в цьому прикладі для кореневої програми визначено лише один маршрут – за допомогою методу app.get вказується обробник для маршрутизації '/'. У процесорі формується відповідь на запит, в який записується вміст файлу index.html, що знаходиться в тій же папці, що і index.js. Використовуючи метод бібліотеки Socket.IO, встановлено обробник події «надсилати повідомлення», який сповіщає всіх підключених клієнтів, генеруючи подію «отримати повідомлення», передаючи дані повідомлення в параметрі msg.

Для розділення html і css недостатньо просто виділити вміст контейнера style в окремий файл, підключений за допомогою тега link (наприклад ). Для надання статичних файлів, таких як зображення, CSS і JavaScript, у фреймворку Express використовується функція проміжної обробки express.static, в яку необхідно передати ім'я каталогу, в якому знаходяться статичні ресурси. У наведеному прикладі файлу index.js, для використання файлом index.html файлу style.css, потрібно внести такі зміни:

var express = require("express"); // окрема змінна для зручності
var app = express();

var http = require('http').Server(app);
var io = require('socket.io')(http);

app.use(express.static('public')); // вказівка каталогу для статичних ресурсів, в якому буде розташований файл css, що підключається.
app.get...

При надсиланні даних із форми (тексту повідомлення) породжується подія 'send message', на яку реагує серверна частина програми. Також у клієнті визначено його власного обробника події 'receive message', що породжується сервером. Для перевірки роботи програми слід відкрити кілька її копій у різних вкладках, імітуючи використання кількома учасниками чату з одним сервером.

3. Опис лабораторної установки

Для виконання лабораторної роботи рекомендується використовувати:

  • текстовий редактор, що має підсвічування синтаксису: Notepad++ або Visual Studio Code;
  • серверна платформа Node.js v10.15.3 LTS;
  • пакет socket.io.

Засоби, що рекомендуються, є вільним програмним забезпеченням.
