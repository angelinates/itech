Лабораторная работа №1

Основы построения и публикации HTML-документов
1.1 Цель работы
  1. Изучить принципы построения HTML-документа, его структуру и форматирование. Рассмотреть возможности форматирования текста, организации списков, использование гиперссылок, форм и графики в WEB-документах. Понятие табличного представления данных.
  2. Познакомиться с системой контроля версий (СКВ) Git и возможностями сервиса Github Pages.
1.2 Описание лабораторной установки
    Для выполнения лабораторной работы рекомендуется использовать следующее свободное программное обеспечение:
  • редактор кода MS Visual Studio Code или любой другой, знакомый пользователю и имеющий подсветку синтаксиса (Notepad++, Sublime Text, Atom, Brackets и т.п.);
  • СКВ Git (для работы с сервисом GitHub Pages потребуется доступ к сети Интернет).
1.3 Порядок выполнения работы
  1. Создать HTML-документ на произвольную тематику на основе произвольного макета страницы (одноколоночного, 2-х или 3-х колоночного, пример структуры страницы с двумя колонками). Основной контент страницы должен содержать блочные и строковые элементы, заголовки. Содержание произвольное. Можно использовать табличную верстку ‐ в целях лучшего освоения HTML и понимания недостатков этого способа построения страницы. В целом, данную лабораторную работу выполнять пока без использования CSS. Применение семантической вёрстки вместо табличной (по желанию!), в случае простой одноколоночной структуры страницы, возможно и без управления размещением блоков средствами CSS.
      2. Согласно варианту, обязательно использовать в созданном документе:
    • фрагмент оформленного текста (строго по указанному в варианте его изображению), используя теги для полужирного выделения текста, выделения курсивом, под- и надстрочного начертания, а также используя &-сущности для специальных символов (стрелок, символов греческого алфавита, угловых скобок и т.п.);
    • список;
    • таблицу;
    • форму с управляющими элементами;
    • гипертекстовые ссылки и изображения.
  2. Воспользоваться сервисом Github Pages (см. указания ‐ с момента их создания интерфейс гитхаба обновился, но сами настройки и их порядок неизменны). Для работы с локальным репозиторием и взаимодействия с удалённым репозиторием на Github использовать консоль (cmd, PowerShell, Git Bash или их аналоги, позволяющие без подсказок использовать команды Git). При получении отработки возможны вопросы по работе с репозиторием.
  3. Проверить отображение созданного веб-ресурса в различных браузерах.
  4. Проверить созданный документ на валидность с помощью онлайн-сервиса, в котором использовать URL-адрес опубликованного на Github Pages документа.
  5. Внести в проект указанные преподавателем (при попытке получения отработки) изменения.
Задания для лабораторной являются индивидуальными. Возможно совместное выполнение в бригаде вариантов всех ее участников.
Номер варианта студента указан в электронном журнале. Выполнение чужого варианта является основанием для отклонения работы.
Отработка выставляется после совместной со студентом проверки корректности работы проекта, ответов студентом на вопросы по ходу работы и (опционально) получения преподавателем ссылки на GitHub репозиторий проекта.

1.5 Содержание отчета
Отчет по лабораторной работе должен содержать: цель работы, вариант задания, эскиз макета страницы, исходный код, изображение полученного документа, выводы по работе. Наличие отчета не является необходимым.
