Лабораторная работа №3

Язык создания сценариев JavaScript
3.1 Цель работы
Изучить язык программирования Java Script для создания интерактивных HTML-документов. Рассмотреть применение библиотеки jQuery.

3.3 Описание лабораторной установки
    Для выполнения лабораторной работы рекомендуется использовать следующее свободное программное обеспечение:
  • редактор MS Visual Studio Code или текстовый редактор Notepad++, имеющий подсветку синтаксиса;
  • браузер Mozilla Firefox или Google Chrome;
  • СКВ Git (для работы с сервисом GitHub Pages потребуется доступ к сети Интернет).
3.4 Порядок выполнения работы
  1. Создать простой HTML-документ (если по заданию не предполагается написание букмарклета).
  2. Добавить скрипт (или подготовить букмарклет) согласно выбранному варианту задания. Возможно применение библиотеки jQuery.
  3. Проверить работу в разных браузерах.
Задания для лабораторной являются индивидуальными. Возможно совместное выполнение в бригаде вариантов всех ее участников. Номер варианта студента рассчитывается через остаток от деления его номера в журнале академгруппы на количество доступных вариантов. В данной лабораторной работе, номер своего варианта можно узнать как N%10, где N ‐ номер в журнале академической группы.
Отработка выставляется после совместной со студентом проверки работы проекта и получения преподавателем ссылки на GitHub/Bitbucket репозиторий проекта, выполненного в соответствии со всеми заданиями этого раздела.
3.5 Варианты заданий

  1. В папке images вашего ресурса находятся изображения 1.jpg, 2.jpg, 3.jpg, 4.jpg, 5.jpg, 6.jpg, 7.jpg, 8.jpg, 9.jpg. На странице выводится первое изображение из этой папки, присутствуют управляющие элементы, позволяющие менять параметры изображения (ширина, высота, толщина рамки, альтернативный текст). По нажатию на изображение менять его на случайное из папки images.

  2. Имеется страница с данными о температуре. Написать скрипт (можно в виде букмарклета), выводящий информацию о самом жарком/холодном дне.

  3. На странице представлена галерея рисунков и кнопки "выше" и "ниже" для изменения их порядка. Написать сценарий, который реализует смену порядка рисунков в документе, а также предоставить возможность изменять параметры изображений (ширину и высоту).

  4. Реализовать смену изображений на странице через заданный пользователем интервал. Оформление страницы (например, фон) должно меняться в зависимости от текущего времени суток.

  5. Написать скрпит, случайным образом генерирующий данные и выводящий их пользователю в виде таблицы. Должно отличаться оформление четных и нечетных строк данных, а также первой строки таблицы (заголовочной). Перед началом вывода пользователь может указать интервал, с которым появляются новые строки с данными.

  6. Написать скрипт для создания таблицы 10 на 10, заполненную случайными числами в диапозоне, указанном пользователем. Оформление таблицы (фон ячеек) по подобию шахматной доски.

  7. Написать сценарий определения числа студентов в каждой из категорий: "отличник", "хорошист" и "неуспевающий". В анкете для каждого студента группы приводится информация: Фамилия и пять оценок за сессию. К категории "отличник" относятся студенты имеющие все оценки "5", к категории "хорошист" относятся студенты имеющие оценки "5" и "4", к категории "неуспевающий" относятся студенты, имеющие хотя бы одну оценку "2". Необходимо последовательно указать данные для произвольного числа студентов, после чего вывести сводку о количестве студентов в каждой из категорий и перечислить их. Для упрощения отладки рекомендуется добавить возможность не заполнять оценки вручную, а использовать генератор случайных чисел.

  8. Отобразить рисунок, согласно выбранному пункту выпадающего списка. Предоставить возможность изменять параметры изображения (ширину и высоту, толщину рамки, альтернативный текст).

  9. На странице находится набор полей для ввода данных о сотрудниках ‐ фамилии и дата приема на работу. Рассчитать и вывести пользователю средний стаж работы.

  10. Калькулятор.

3.6 Содержание отчета
Отчет по лабораторной работе должен содержать: цель работы, пункты заданий из порядка выполнения работы, исходный код веб-страницы, стилевых файлов и файлов скриптов, изображение полученного документа, выводы по работе. Для защиты наличие распечатанного отчета не является необходимым.
