Лабораторна робота №4

Основи мови PHP
4.1 Мета роботи
  • Вивчити мову програмування PHP для розробки динамічних web-документів.
  • Ознайомитись з використанням збірок для веб-розробки на основі веб-сервера Apache.
4.2 Необхідні засоби
Для виконання лабораторної роботи рекомендується використовувати таке відкрите програмне забезпечення:
  • редактор коду MS Visual Studio Code або будь-який інший, який знайомий користувачу і забезпечує підсвічування синтаксису (Notepad++, Sublime Text, Atom, Brackets и т.п.);
  • збірка для веб-розробки OpenServer;
  • браузер Mozilla Firefox або Google Chrome;
  • СКВ Git (для роботи з сервісом GitHub потрібен доступ до мережі Інтернет).
4.3 Порядок виконання роботи
  1. Переконатися, що веб-сервер працює належним чином. Наприклад, створивши найпростішу сторінку з викликом функції phpinfo() та подивившись на неї у браузері, надіславши запит до веб-сервера. Шлях за замовчуванням (якщо не налаштовувати свої домени) для розміщення своїх проектів у каталозі встановленого OpenServer - domains\localhost (є зручний доступ через ПКМ на іконці опенсервера в системному треї). Рекомендується не розміщувати файли лабораторної роботи у цій папці, а створити у ньому свій підкаталог, наприклад, lab4, у якому працювати у файлі index.php. Тоді запит у браузері буде http://localhost/lab4/index.php
  2. Розробити web-застосунок відповідно до варіанта завдання.
Завдання для лабораторної роботи є індивідуальними. Можливе спільне виконання у бригаді варіантів усіх її учасників.
Номер варіанта студента вказано в електронному журналі. Виконання чужого варіанту є підставою для відхилення роботи.
Відпрацювання виставляється після спільної зі студентом перевірки роботи проекту, виконаного відповідно до всіх завдань цього розділу.
4.4 Варіанти завдань

  1. Написати скрипт підрахунку кількості показів сторінки. Кількість відвідувань зберігатиметься в текстовому файлі. Використовувати функції для роботи з файлами (наприклад, fread, fwrite, fclose...).

  2. Написати сценарій проходження форми логіну. Існує форма з полями "логін", "пароль", якщо введені дані не збігаються з правильними (визначеними в текстовому файлі), то виводити повідомлення про помилку, якщо збігаються, то виводити "ви залогінені". Використовувати масив глобальних змінних $_POST (метод відправки форми, відповідно, теж POST).

  3. Реалізувати сценарій голосування. Для цього знадобляться три файли: файл, який містить форму для голосування; файл, куди відвідувач буде перенаправлятися (з відображенням накопичених результатів голосувань); текстовий файл для збереження результатів голосування.

  4. Створити сайт, вміст сторінок якого братиметься з текстових файлів. Зробити можливість додавання та редагування вмісту цих текстових файлів.

  5. Написати сценарій виведення у вигляді таблиць інформації, взятої з файлу. Передбачити формат зберігання такої інформації, щоб її було зручно відображати як таблицю. Рекомендується познайомитися з форматом CSV (але ніхто не заважає використовувати і свої довільні роздільники).

  6. Написати сценарій калькулятора, який виконує найпростіші арифметичні дії. Вести історію використання калькулятора у текстовому файлі та виводити її на стартовій сторінці.

  7. На першій сторінці сайту розмістити форму для введення імені користувача. При переході на інші сторінки виводити це ім'я. Використовувати сесії та кукіси для цього (порівняти використання цих механізмів). Після того, як користувач надіслав "ім'я користувача", зареєструвати нову змінну в cookies зі значенням цього імені та зареєструвати нову змінну в сесії зі значенням цього імені. При переході на інші сторінки виводити це ім'я. Використовувати функції для роботи з сесіями та cookies та масиви глобальних змінних $_COOKIE та $_SESSION.

  8. Створити два довільні цілочисленні масиви. Написати сценарій, який:
    • прибирає всі елементи, що дублюються, з кожного масиву;
    • виводить кількість дублюючих елементів двох масивів;
    • зливає два масиви в один, прибираючи всі значення, що дублюються;
    • змінює місцями значення масиву (перший елемент масиву стає останнім, передостанній – другим, …, останнім першим). Використовувати для цього цикл foreach (замість вже наявної функції reverse).
4.5 Зміст звіту
Звіт з лабораторної роботи повинен містити: мету роботи, варіант завдання, вихідний код веб-сторінки, стильових файлів та файлів скриптів, зображення отриманого документа, висновки по роботі. Наявність звіту не є обов'язковою.
