Лабораторная работа №4

Основы языка PHP
4.1 Цель работы
  • Изучить язык программирования PHP для разработки динамических Web-документов.
  • Познакомиться с использованием сборок для веб-разработки на основе веб-сервера Apache.
4.2 Описание лабораторной установки
    Для выполнения лабораторной работы рекомендуется использовать следующее свободное программное обеспечение:
  • редактор кода MS Visual Studio Code или любой другой, знакомый пользователю и имеющий подсветку синтаксиса (Notepad++, Sublime Text, Atom, Brackets и т.п.);
  • сборка для веб-разработки OpenServer;
  • браузер Mozilla Firefox или Google Chrome;
  • СКВ Git (для работы с сервисом GitHub потребуется доступ к сети Интернет).
4.3 Порядок выполнения работы
  1. Удостовериться в корректной работе веб-сервера. Например, создав простейшую страничку с вызовом phpinfo() и посмотрев на неё в браузере, отправив запрос к веб-серверу. Путь по умолчанию (если не настраивать своих доменов) для размещения своих проектов в каталоге установленного Openserver - domains\localhost (есть удобный доступ через ПКМ на иконке опенсервера в системном трее). Рекомендуется не размещать файлы лабораторной работы непосредственно в этой папке, а создать в нём свой подкаталог, например, lab4, в котором вести работу в файле index.php. Тогда запрос в браузере будет http://localhost/lab4/index.php
  2. Разработать Web-приложение в соответствии с вариантом задания.
Задания для лабораторной являются индивидуальными, но при работе в составе бригады, возможно совместное выполнение в бригаде вариантов всех ее участников.
Номер варианта студента указан в электронном журнале. Выполнение чужого варианта является основанием для отклонения работы.
Отработка выставляется после совместной со студентом проверки работы проекта, выполненного в соответствии со всеми заданиями этого раздела.
4.4 Варианты заданий

  1. Написать скрипт подсчета количества показов страницы. Количество посещений хранить в текстовом файле. Использовать функции для работы с файлами (например fread, fwrite, fclose...).

  2. Написать сценарий прохода формы логина. Есть форма с полями "логин", "пароль", если введенные данные не совпадают с правильными (определенными в текстовом файле), то выводить сообщение об ошибке, если совпадают, то выводить "вы залогинены". Использовать массив глобальных переменных $_POST (метод отправки формы - POST).

  3. Написать скрипт для голосования. Для этого понадобятся три файла - файл, содержащий форму для голосования; файл, куда проголосовавший посетитель будет перенаправляться (с отображением накопленных результатов голосований); и текстовый файл для хранения результатов голосования.

  4. Создать сайт, содержимое страниц которого будет браться из текстовых файлов. Сделать возможность добавления и редактирования содержимого этих текстовых файлов.

  5. Написать скрипт вывода в виде таблиц информации, взятой из файла. Предусмотреть формат хранения такой информации, чтобы её было удобно отображать в виде таблиц. Рекомендуется познакомиться с форматом CSV (но никто не мешает использовать и свои произвольные разделители).

  6. Написать скрипт калькулятора, выполняющего простейшие арифметические действия. Вести историю использования калькулятора в текстовом файле и выводить её на стартовой странице.

  7. На первой страничке сайта разместить форму для ввода имени пользователя. При переходе на другие странички - выводить это имя. Использовать сессии и кукисы для этого (сопоставить использование этих механизмов). После того как пользователь отправил "имя пользователя", зарегистрировать новую переменную в cookies со значением этого имени и зарегистрировать новую переменную в сессии со значением этого имени. При переходе на другие странички - выводить это имя. Использовать функции для работы с сессиями и cookies и массивы глобальных переменных $_COOKIE и $_SESSION.

  8. Создать два произвольных целочисленных массива.
      Написать скрипт, который
    • убирает все дублирущие элементы каждого массива;
    • выводит количество дублирущих элементов двух массивов;
    • сливает два массива в один, убирая все дублирующиеся значения;
    • меняет местами значения массива (первый элемент массива становится последним, предпоследний - вторым, ..., последний первым). Использовать foreach.
4.5 Содержание отчета
Отчет по лабораторной работе должен содержать: цель работы, вариант задания, исходный код веб-страницы, стилевых файлов, изображение полученного документа, выводы по работе. Наличие отчета не является необходимым.
