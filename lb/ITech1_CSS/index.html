Лабораторная работа №2

Каскадные таблицы стилей
2.1 Цель работы
  • Изучить принципы встраивания каскадных таблиц стилей в документ, разновидности селекторов и особенности их использования, познакомиться с распространёнными стилевыми свойствами.
  • Изучить способы позиционирования элементов.
2.2 Описание лабораторной установки
    Для выполнения лабораторной работы рекомендуется использовать следующее свободное программное обеспечение:
  • редактор кода MS Visual Studio Code или любой другой, знакомый пользователю и имеющий подсветку синтаксиса (Notepad++, Sublime Text, Atom, Brackets и т.п.);
  • браузер Mozilla Firefox или Google Chrome;
  • СКВ Git (для работы с сервисом GitHub потребуется доступ к сети Интернет).
2.3 Порядок выполнения работы
  1. Создать HTML-документ с произвольным содержимым, на котором в ходе лабораторной работы в дальнейшем будет демонстрироваться использование CSS. Имеет смысл сразу предусмотреть использование как блочных элементов, так и строковых, для будущего удобства.
  2. Показать в созданном документе такие способы использования каскадных таблиц стилей:
    • inline-описание (используется атрибут style выбранного вами элемента);
    • с помощью контейнера <style> в секции заголовка;
    • подключить стилевую таблицу, расположенную во внешнем CSS-файле.
  3. Используя селекторы тегов, классов и идентификаторов, показать использование стилей (например, для форматирования текста, изменения фона элемента и т.д.). Использование, помимо указанных, других видов селекторов и их комбинаций может пригодиться для упрощения поиска ответов на вопросы по коду при получении отработки или в ходе защиты.
  4. Добавить в документ элемент, для стилизации которого использовать свойство position и свойства для указания смещения положения элемента. Использовать абсолютное, относительное или фиксированное позиционирование.
  5. Добавить в документ элементы, для стилизации которых использовать свойство float. Удобно показывать данное свойство на примере расположения навигационных элементов, или при обтекании текстом изображений.
  6. Добавить в документ элементы согласно варианту.
  7. Быть готовым к вопросам по коду.
  8. Проверить отображение созданного документа в различных браузерах.
  9. Внести в проект указанные преподавателем (при попытке получения отработки) изменения.
Задания для лабораторной являются индивидуальными, но при работе в составе бригады, возможно совместное выполнение в бригаде вариантов всех ее участников.
Номер варианта студента указан в электронном журнале. Выполнение чужого варианта является основанием для отклонения работы.
Отработка выставляется после совместной со студентом проверки работы проекта, выполненного в соответствии со всеми заданиями этого раздела.
2.4 Варианты заданий
Обратите внимание, что задания по вариантам являются лишь частью работы. Внимательно читайте порядок выполнения!
  1. Добавить на страницу несколько флажков (элементов checkbox), оформление которых (цвет фона метки, например) зависит от их состояния.
    Добавить на страницу контейнер с элементами, размещёнными согласно рисунку. Использовать Flexbox.
  2. Добавить на страницу переключатели (элементы radio), оформление которых зависит от их состояния.
    Добавить на страницу контейнер с элементами, размещенными согласно рисунку. Обратите внимание, в HTML-коде элементы размещены по порядку "Я первый", "Я второй", "Я третий", "Я четвёртый". А с помощью Flexbox и CSS-свойства order, изначальный порядок должен быть изменён на тот, который соответствует приведенному изображению.
  3. Добавить на страницу список с собственным маркером (изображение, символ или сгенерированное средствами CSS).
    Добавить на страницу контейнер с элементами, размещенными согласно рисунку. Использовать Flexbox.
  4. Добавить на страницу форму регистрации пользователя, оформление полей которой зависит от корректности заполнения (valid/invalid) и обязательности. Форма должна быть центрирована по горизонтали и "не прокручиваться", т.е. позиция не должна меняться при скроллинге страницы.
  5. Добавить на страницу скрытый блок, который не будет изменять свою позицию при прокрутке страницы, и у которого видна лишь часть, по наведению на которую блок разворачивается полностью. Пример на рисунке.
  6. Используя средства CSS, добавить на страницу раскрывающийся спойлер (accordion). Пример на рисунке.
  7. Добавить на страницу изображение, размер которого будет варьироваться в зависимости от размера окна браузера: при max-width: 1200px, ширина изображения составляет 600px, а при max-width: 800px - всего 400px. Использовать директиву media.
    Добавить на страницу таблицу, центрировать её по горизонтали. Таблица должна быть оформлена согласно примеру. Обязательно использовать псевдоклассы.
  8. Добавить на страницу элемент, содержащий переключаемые вкладки. Пример на рисунке.
  9. Добавить на страницу вертикальное навигационное меню на основе списка ссылок. Стилизация предполагает изменение цвета фона для текущей ссылки (предположим, первой в списке) и ссылки, на которую пользователь наводит курсор мыши. Пример на рисунке.

    Добавить на страницу прыгающую кнопку, используя средства CSS-анимации. Пример на рисунке.
2.5 Содержание отчета
Отчет по лабораторной работе должен содержать: цель работы, вариант задания, эскиз макета страницы, исходный код, изображение полученного документа, выводы по работе. Наличие отчета не является необходимым.
