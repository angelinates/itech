Лабораторная работа №1 (2018)

Изучение расширения PDO для обеспечения абстракции доступа к базам данных
1.1 Цель работы
Изучение методов управления подключениями, выполнения запросов и транзакций в расширении PDO для реализации основных возможностей различных СУБД в WEB приложении.
1.2 Методические указания

При подготовке к выполнению лабораторной работы следует повторить синтаксис операторов языка SQL применительно к операциям выборки (SELECT) и модификации (INSERT, DELETE, UPDATE) данных.

Кроме того, требуется ознакомиться с программными методами выполнения запросов к базе данных, а также этапами выполнения подготовленного запроса к базе данных с применением интерфейса PDO [5,6].

1.3 Описание лабораторной установки
Для выполнения лабораторной работы рекомендуется использовать:
  • текстовый редактор, имеющий подсветку синтаксиса: Notepad++ или Visual Studio Code;
  • браузер: Mozilla Firefox или Google Chrome;
  • сборка веб-сервера: XAMPP или OpenServer.
Рекомендуемые средства являются свободным программным обеспечением.
Для выполнения лабораторной работы не требуется доступ к сети Интернет.
1.4 Порядок выполнения работы
  1. Используя утилиту phpMyAdmin, выполнить импорт БД по вашему варианту.
  2. Подготовить требуемые по варианту SQL-запросы к БД.
  3. Проверить выполнение запросов в phpMyAdmin, используя произвольные параметры.
  4. Создать веб-приложение, запрашивающее у пользователя параметры для запросов и выводящее их результаты. Рекомендуется:
    • разделить исходный код на стартовую страницу (на которой расположены формы для указания параметров запросов), и отдельные файлы для обработчиков каждого запроса.
    • Код для создания подключения имеет смысл разместить в отдельном файле, и подключать его по месту требования.
    • Элементы управления в формах для ввода параметров, где возможно, стоит делать не текстовыми полями, а выпадающими списками (формируя их на основе данных из БД) - это позволит пользователю выбирать возможные варианты данных для запроса, а не вводить их самому.
Окончательный вариант задания демонстрируется в электронном виде.
Возможно выполнение нестандартного задания. В этом случае тема задания должна быть предварительно согласована.
